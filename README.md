# README

## Overview
This repository provides a comprehensive workflow designer meant for human resources processes.

## Architecture
The architecture is designed to effectively manage the data flows between different components. The following diagram illustrates the data flow:

![Data Flow Diagrams](link_to_data_flow_diagram.png)

## Design Decisions
The technology choices made for this project include:
- **Framework**: We chose React for the frontend due to its component-based architecture, enabling reusability and maintainability.
- **Backend**: Node.js was selected for its non-blocking architecture, allowing for real-time data processing with high concurrency.
- **Database**: MongoDB was utilized for its flexibility in handling varying data structures. 

## What Was Completed
### Features
- **User Authentication**: Implemented OAuth2 for secure authentication.
 - **Data Visualization Tooling**: Created integrated graphing solution for user data.

### Challenges Overcome
- **Data consistency**: Achieved through transactions and careful architecture design.
- **Response latency**: Optimized backend APIs for faster data retrieval.
